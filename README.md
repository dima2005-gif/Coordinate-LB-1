# Лабораторна робота №1

## Програмні моделі систем координат
### Мета
- Спроектувати та реалізувати імутабельні програмні моделі для представлення точок у 2D та 3D системах координат.
- Реалізувати механізми перетворення між декартовою, полярною та сферичною системами координат з використанням статичних фабричних методів.
- Навчитись обчислювати відстані між точками, використовуючи різні математичні підходи.
- Провести аналіз продуктивності обчислень для різних представлень даних.

# Інструкція для запуску
Дана лабораторна робота виконана мовою програмування JavaScript(JS). Для запуску необхідний **Node.js**.
Для перевірки у термінал пишемо команду:

```js
node --version
```
Якщо **Node.js** не встановлена, то завантажемо его з офіційного [сайту](https://nodejs.org/en/download)

# Аналізи та висновки
### 2D
Теоретично полярна формула повинна бути повільнішою через операцію `cos`. Однак на практиці після прогріву вона виявилась швидшою (1.657 мс проти 2.311 мс).
Причина — V8 добре оптимізує вбудовані математичні функції (`Math.cos`) на рівні нативних інструкцій CPU, тоді як евклідова формула читає більше полів з пам'яті (`x1`, `y1`, `x2`, `y2`), що створює додаткове навантаження.

### 3D
Найшвидший метод — хорда (2.337 мс), найповільніший — евклід (5.896 мс). Дуга займає проміжне місце через `Math.acos`, який є дорожчою операцією ніж `Math.cos`. Евклід знову програє з тієї ж причини — доступ до шести полів об'єктів (`x`, `y`, `z` у двох точках) менш ефективний
